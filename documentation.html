Red-Headed Pro v1.4

Export your WooCommerce orders everywhere, anytime — multi-format, multi-destination, by hand or on a schedule, or the moment an order changes status.

Introduction

Red-Headed Pro builds a file from your orders, then delivers that file where you want it. The format, the columns, the orders included, and the destination are all configured per profile. Run as many profiles in parallel as you need — one for accounting, one for the shipping carrier, one for a marketplace partner.

Build

Six file formats: CSV, TSV, JSON, NDJSON, XML, XLSX. Pick columns and order them visually. Add custom meta fields.

Filter

Status, date range, payment method. Preview the matches before running.

Deliver

Email, SFTP, Google Drive, REST endpoint, Local ZIP, or a one-click download.

Automate

Manual, hourly, daily, weekly, or fired the moment an order changes status.

ℹ️
Built on top of WooCommerce. Red-Headed reads your orders directly — no duplication, no sync delay. Compatible with High-Performance Order Storage (HPOS) and WPML / Polylang out of the box.

Installation & Requirements

Requirements

  • WordPress 6.2+ with PHP 8.1+
  • WooCommerce 8.0+
  • The Froggy Hub plugin (required — provides the license manager and shared framework)

Installation steps

1

Install The Froggy Hub

Upload and activate the Froggy Hub plugin first. Red-Headed Pro will not load without it.

2

Upload Red-Headed Pro

Go to Plugins → Add New → Upload Plugin and upload the .zip file. Activate it.

3

Activate your license

Open The Froggy Hub → Licenses. Paste your license key. The Red-Headed dashboard appears as soon as the license is validated.

⚠️
Lite vs Pro. If Red-Headed Lite is active, it is deactivated automatically when Pro loads. No data is lost — profiles and history carry over.

Your first export

Three steps to a working export.

1

Create a profile

Open Red-Headed Pro → Settings → Profiles and click + New profile. Name it (e.g. Daily completed) and pick a format. CSV is the safe default.

2

Pick filters and a destination

Tick the order statuses you want (start with all, narrow later). Add a destination — Email is the fastest to test.

3

Run it

Save the profile, then click ▶ Run. The job appears in the Exports tab. Open it to download the file or preview the first rows.

💡
Preview before you run. Click the 👁️ icon next to ▶ Run to see which orders match the profile filters. No job is created, no file is saved.

Export profiles

A profile is a saved recipe combining four ingredients:

  • Format — CSV, TSV, JSON, NDJSON, XML, or XLSX.
  • Filters — which orders to include.
  • Columns — which fields to put in the file.
  • Destinations — where to send the file.

Optional add-ons: schedule (recurring) and auto-trigger (event-driven on order status change).

ℹ️
Profiles are independent. Run as many as you need: one for accounting, one for shipping, one per marketplace.

Filters

Three filters narrow which orders end up in the file.

FilterWhat it doesRecommended start
StatusesOnly orders matching one of the ticked statuses are included.Tick all to start. Narrow later (e.g. Completed only for accounting).
Date from / toFilters by order creation date.Leave both blank for "all time", or set Date from only to capture from a cut-off.
Payment methodFilters by payment gateway slug (e.g. stripe, paypal).Leave blank to include all payment methods.
⚠️
Empty file? The most common cause is a filter that excludes every order. The Exports list shows a "⚠ check filters" badge next to the 0-record line. Click 👁️ Preview on the profile to see what matches and adjust.

Columns

The column picker is where you choose which fields end up in the exported file. Open it from the profile editor: + Browse fields.

1

Open the picker

The picker opens centered with all available fields grouped by category — order, customer, billing, shipping, totals, items, custom fields.

2

Tick the fields you want

Each field appears in the active list as you tick it. Untick to remove.

3

Reorder and rename

Drag active columns to reorder. Click the column header label to rename it for the exported file.

💡
Custom meta fields. The Meta group at the bottom of the picker lets you add any order custom field by key — e.g. _vat_number, _shipping_carrier.

Destinations

A destination is where the file goes once it is built. A profile can stack many destinations — the same file is delivered to all of them in one run.

TypeRequired fieldsNotes
EmailRecipient emailThe file is attached. Use a comma to send to multiple recipients.
SFTPHost, port, user, password, pathOptional filename pattern. The remote folder must exist and be writable.
Google DriveOAuth access_token, optional folder IDTokens last roughly 1 hour. Full OAuth flow with auto-refresh ships in a future release.
RESTURL, auth type, token / credentialsPOSTs the file as a multipart upload. Bearer, Basic, or custom header auth.
Local ZIPNoneThe file is also saved as a ZIP archive in wp-content/uploads/lion-frog/exports/.
DownloadNoneThe Exports tab shows a one-click download link.
ℹ️
Stacking destinations. Add the same file to Email + SFTP + Google Drive in one profile — everyone gets a copy in one run.

Filename pattern

Customize the name of the delivered file using placeholders. Available on Google Drive and SFTP destinations. An empty pattern keeps the auto-generated filename.

Available placeholders

PlaceholderResolves to
{profile}Profile name
{format}csv, tsv, json, ndjson, xml, xlsx
{records}Number of rows in this export
{job_id}Internal job ID (useful for log correlation)
{date}Y-m-d
{time}H-i-s
{datetime}Y-m-d_H-i-s
{timestamp}Unix epoch seconds
{order_id}First order ID
{order_number}First order number (e.g. e-4123)
{customer_id}First order customer ID
{customer_email}First order billing email
{customer_name}First order billing first + last name
{random}6-character alphanumeric (uniqueness)
💡
Useful patterns. Single-order exports: order-{order_number}-{date}.{format}. Daily batch: orders-{date}.{format}. Carrier feed: {profile}-{datetime}.{format}.

Schedule

Run a profile automatically on a recurring cycle.

ScheduleCycle
Manual onlyThe profile runs only when you click ▶ Run.
HourlyEvery hour, at the top of the hour.
DailyOnce per day.
WeeklyOnce per week.
MonthlyOnce per month.
⚠️
WordPress cron caveat. Schedules use the WordPress cron, which only fires on traffic. On a low-traffic site, expected runs may slip by minutes or hours. For precise timing, disable WP-Cron and call wp-cron.php from a system cron once a minute.

Auto-trigger (on status change)

Fire the export the moment an order moves to a specific status. This runs on top of (and independently from) the schedule.

Setup

1

Edit the profile

Scroll to the Auto-trigger block.

2

List the trigger statuses

Type one or more order statuses, comma-separated — e.g. completed or completed, processing.

3

Set guards (optional)

Min total ignores orders below the threshold. Fire only once per order avoids duplicates if the order's status flaps.

Exports list

The Exports tab is the activity log of every run. Each row shows the format, record count, file size, what triggered it, when it started, how long it took, and whether it succeeded.

Order tracking

Every order remembers when it was last exported and how many times.

Where to see it

Open WooCommerce → Orders. The new 📦 Exported column shows:

  • — (gray) if the order has never been exported.
  • ✓ 3× (green) if the order has been exported three times. Hover for the date of the last export.
💡
Audit-friendly. The Exported column lets the operations team confirm at a glance which orders have already shipped to the accounting / shipping / partner pipeline.

Preview

Two preview surfaces, both opened by clicking the 👁️ icon.

WhereWhat it shows
On a profile (next to ▶ Run)Dry-runs the filters and shows the first 5 orders that would be exported, with the columns mapped. No job is created, no file saved.
On a job in the Exports listReads the saved file inline and shows the first 10 rows. Lets you spot-check the content without downloading.

Troubleshooting

SymptomLikely causeFix
Export ran, file is empty (only headers) Filters exclude every order — most often the date window or the status list. Click 👁️ Preview on the profile. Widen the date range or tick more statuses.
SFTP fails with "SFTP authentication failed" Wrong password or the user lacks permission on the target folder. Test the credentials in a desktop SFTP client. Confirm the remote folder exists and is writable.
Google Drive upload returns 401 Unauthorized The access token expired (tokens last roughly 1 hour). Generate a fresh token from Google's OAuth Playground and paste it into the destination.
Email is sent but the recipient never gets it The site has no SMTP configured, so the message lands in spam or is dropped. Install an SMTP plugin (e.g. WP Mail SMTP) and configure a transactional sender.
The schedule never fires WordPress cron only runs on traffic. Low-traffic sites can wait minutes or hours. Disable WP-Cron and call wp-cron.php from a system cron once a minute.
XLSX export fails with "ZipArchive missing" The host's PHP build is missing the ZipArchive extension. Ask the host to enable ZipArchive, or switch the profile format to CSV / JSON in the meantime.

Performance & limits

  • Order volume. Exports stream row by row. Tens of thousands of orders work; hundreds of thousands take minutes — schedule those overnight.
  • Memory. XLSX is the heaviest format because the spreadsheet engine builds in memory. CSV / JSON / NDJSON are much lighter — prefer them for large exports.
  • HPOS. Compatible with WooCommerce High-Performance Order Storage. No configuration needed.
  • WPML / Polylang. Compatible. Order language is preserved in the export.
  • File retention. Generated files live in wp-content/uploads/lion-frog/exports/. Old files are not auto-deleted — clean up periodically if disk space matters.

REST API

Endpoints under /wp-json/pelican/v1/:

  • POST /profiles — create / update a profile.
  • GET /profiles/{id} — read a profile.
  • POST /run/{profile_id} — trigger an export (returns the job ID).
  • GET /jobs — list past jobs with filters.
