Red-Headed Pro v1.5

Export your WooCommerce orders everywhere, anytime — multi-format, multi-destination, by hand or on a schedule, or the instant an order changes status.

Introduction

Red-Headed Pro builds a file from your WooCommerce orders and delivers it where you need it. The format, the columns, the orders included, and the destination are all configured per profile. Run as many profiles in parallel as you need — one for accounting, one for the shipping carrier, one for an ERP feed.

Build

Six file formats: CSV, TSV, JSON, NDJSON, XML, XLSX. Pick columns visually, rename headers, add custom meta, computed fields.

Filter

14 filters: status, date range, payment method, SKU, category, country, city, customer role, coupon, total range, custom meta…

Deliver

Seven destinations: Email, SFTP, Google Drive, REST endpoint, Local folder, Local ZIP, direct download.

Automate

Manual, hourly, daily, weekly, monthly — or fired the moment an order changes status, with deduplication.

ℹ️
Built on WooCommerce. Red-Headed reads orders directly from WooCommerce (HPOS compatible). No duplication, no sync delay. Works with WPML, Polylang, and custom order statuses out of the box.

1

Upload and activate the Hub first. Red-Headed Pro will not load without it.

2

Upload Red-Headed Pro

Go to Plugins → Add New → Upload Plugin and upload the .zip file. Activate it.

3

Activate your license

Open The Froggy Hub → Licenses. Paste your license key. The Red-Headed dashboard appears once the license is validated.

⚠️
Lite → Pro upgrade. If Red-Headed Lite is active, it is deactivated automatically when Pro loads. No data is lost — profiles, export history, and settings carry over seamlessly.

Your first export

1

Create a profile

Open Red-Headed → Settings → Profiles and click + New profile. Give it a name (e.g. Daily completed) and pick a format — CSV is the safest default.

2

Pick filters and a destination

Tick the order statuses you want (start with all, narrow later). Add at least one destination — Email or Download are the fastest to test.

3

Run it

Save the profile, then click ▶ Run. The job appears in the Exports tab. Open it to download the file or preview the first rows.

💡
Dry-run first. Click the 🧪 Test button next to Run. It builds the file but skips delivery — useful to verify your column mapping and filters before sending anything.

Export profiles

A profile is a saved recipe combining five ingredients:

  • Format — the file type (CSV, JSON…).
  • Filters — which orders to include.
  • Columns — which fields appear in the file, in what order, with what names.
  • Destinations — where to send the finished file.
  • Filename pattern — how to name the delivered file.

Optional layers: schedule (recurring cron), auto-trigger (event-driven on status change), post-export status (mark orders after export), and one-file-per-order split.

ℹ️
Profiles are independent. You can run multiple profiles targeting different audiences: one for accounting, one for the warehouse, one per marketplace partner.

Columns & fields

The column picker controls which data fields appear in your export. Open it from the profile editor.

Using the picker

1

Open the picker

Click + Browse fields. A modal appears with all available fields grouped by category: Order, Customer, Billing, Shipping, Totals, Items, Custom.

2

Tick the fields

Each ticked field is added to the active column list.

3

Reorder and rename

Drag columns to reorder. Click the label to rename the header in the exported file (aliasing).

Custom meta fields

The Meta group at the bottom of the picker lets you add any order meta key as a column (e.g. _vat_number, _shipping_carrier).

💡
Discover meta keys. Click the 🔍 Scan order meta button. It queries your database and lists every meta key in use across all orders. Click + Add next to any key to add it as a column instantly.

Static columns PRO

A static column injects a fixed value on every row. Useful for adding a source identifier, a version tag, or any constant your downstream system expects.

Computed columns PRO

A computed column evaluates a math expression using order data. Available placeholders inside the formula:

PlaceholderDescription
{total}Order total
{subtotal}Subtotal (before tax/shipping)
{tax_total}Total tax amount
{shipping_total}Shipping cost
{discount_total}Discount amount

Example: a "margin estimate" column with formula {subtotal} * 0.35.

JSON shapes & casts PRO

When exporting to JSON, you can choose the structure of the output and force data types on individual columns.

JSON shapes

ShapeDescriptionWhen to use
Flat (default)Array of objects using internal column keys.General-purpose; you control keys via column naming.
LabeledArray of objects using your column labels as keys.Human-readable output; keys match your renamed headers.
NestedLike labeled, but line items are grouped into a sub-array under a configurable key.ERP integrations expecting a parent order with child line items.

Line-items key (nested shape only): controls the name of the sub-array (e.g. items, lines, products).

Bare array (checkbox): removes the wrapper object — the file is a plain JSON array [ {...}, {...} ] instead of { "meta": {...}, "orders": [...] }.

Per-column casts

Each column can be assigned a cast to force the data type in the exported file. This is critical for ERP/JSON integrations that expect strict types.

CastEffectExample input → output
(none)Raw value, no conversion.300 → 300
stringForce string. Booleans become "1" / "".300 → "300"
intCast to integer."1440.5" → 1440
numberCast to float (JSON number)."1440" → 1440.0
money2Format as a 2-decimal money string.1440 → "1440.00"
date:FORMATRe-format a date using any PHP date format.Cast date:d/m/Y on date_created → "29/05/2026"
💡
ERP tip: Many ERP systems need quantities as strings and prices as numbers (or vice-versa). Use casts on the relevant columns to match exactly what the receiving system expects, avoiding import errors.

Destinations

A destination is where the file goes once built. A profile can have multiple destinations — the same file is delivered to all of them in one run (fan-out).

EmailLite (30/day) / Pro (∞)File attached (or notification-only). Configurable subject, sender, CC/BCC.
SFTPLite + ProSecure file transfer. Password encrypted at rest (AES-256-CBC).
Google DriveProUpload to a Drive folder via OAuth token.
REST endpointProPOST/PUT the file to a URL. Bearer, Basic, or custom header auth.

Local ZIPProArchive the file as a ZIP in wp-content/uploads/lion-frog/exports/.
DownloadLite + ProOne-click download link on the Exports page. No configuration needed.

Filename pattern PRO

Customize the delivered file name using placeholders. Available on SFTP, Google Drive, Local folder, and Email destinations. An empty pattern keeps the auto-generated name.

The file extension is auto-appended if missing.

💡
Recipe examples:
Single-order ERP feed: order-{order_id}-{order_datetime}.{format}
Daily batch: orders-{date}.{format}
Carrier feed: {profile}-{datetime}-{random}.{format}
EU date style: export-{date_eu}.{format}
Custom date: export-{date:d_m_Y}.{format}

See the full placeholder reference for every available token.

Email destination

Sends the export file as an email attachment (or a notification-only email if the attachment toggle is unchecked).

ℹ️
Fallback chain. Each field first checks the per-destination value, then falls back to the global default in Settings → Destinations → Email, then to the WordPress default.
⚠️
Deliverability. Emails are sent via wp_mail(). If your host does not have SMTP configured, messages may land in spam or be dropped entirely. Install an SMTP plugin (e.g. WP Mail SMTP) for reliable delivery.

SFTP destination

Uploads the export file to a remote server via SFTP (SSH File Transfer Protocol). Password is encrypted at rest with AES-256-CBC using wp_salt('auth').

Configuration

FieldDescriptionDefault
HostServer hostname or IP.—
PortSSH port number.22

Common pitfalls: (1) The remote folder must exist and be writable. (2) Some servers use non-standard ports (e.g. 2222 or 2223). (3) Uses phpseclib3 by default, with ssh2 extension as fallback.

REST endpoint PRO

POSTs or PUTs the export file to a URL. Ideal for webhook-style integrations, custom APIs, or middleware.

Configuration

FieldDescription
URLThe endpoint URL (HTTPS recommended).
Auth typebearer (Authorization: Bearer token), basic (Authorization: Basic base64), or header (custom header name:value).

The file is sent as raw body content with the appropriate Content-Type header guessed from the format (e.g. text/csv, application/json). HTTP 2xx = success, 4xx/5xx = error.

Local folder & Local ZIP PRO

Local folder

Copies the raw export file (no zip) to a directory on the server. Useful for ERP systems that watch a pickup folder.

  • Relative path: relative to wp-content/ (e.g. order-exports → wp-content/order-exports/).
  • Absolute path: must be inside the WordPress root for security.
  • Empty: defaults to wp-content/order-exports/.

Click Create folder if missing in Settings → Destinations to ensure the folder exists.

Local ZIP

No configuration needed. The export file is also saved as a .zip archive in wp-content/uploads/lion-frog/exports/.

Retry on failure PRO

Enable Retry on fail in the profile to automatically re-attempt delivery on the next cron tick if a destination fails. Set a maximum retry count to avoid infinite loops.

⚠️
WordPress cron caveat. WP cron only fires on page loads. On low-traffic sites, schedules may slip by minutes or hours. For precise timing, add this to wp-config.php:
define( 'DISABLE_WP_CRON', true );
Then set up a system cron to call wp-cron.php every minute.

Auto-trigger (on status change)

Fire an export the instant an order moves to a specific status. This runs independently from the schedule.

1

Edit the profile

Scroll to the Auto-trigger section.

2

Set trigger statuses

List one or more order statuses, comma-separated — e.g. completed or completed, processing.

3

Set guards (optional)

Min total: ignore orders below the threshold. Fire only once per order: prevents duplicate exports if the status flaps back and forth.

💡
Real-time ERP feed. Combine auto-trigger on processing with one-file-per-order split and SFTP delivery to push each order to your ERP the moment it comes in.

Enable Split: one file per order in the profile to produce a separate file for each order in the batch. Each file is delivered individually to all destinations.

This is essential for ERP systems expecting one file per order. Combine with filename placeholders like {order_id} or {order_number} so each file gets a unique name.

Webhooks PRO

Receive real-time HTTP notifications when exports complete, are delivered, or fail. Configure endpoints in Settings → Webhooks.

Events

EventFires when
export.generatedThe export file has been built successfully.
export.deliveredThe file has been delivered to all destinations.
export.failedThe export or a delivery failed.

Configuration per endpoint

  • URL — the endpoint receiving the POST (HTTPS recommended).
  • Secret — optional; used to sign the payload with HMAC-SHA256.
  • Events — pick which events trigger this endpoint.

HMAC signature verification

If a secret is set, every webhook POST includes an X-Red_Headed_Pro-Signature header:

X-Red_Headed_Pro-Signature: sha256=<HMAC-SHA256 of body with your secret>

To verify on your server (PHP example):

// Read raw body and header
$body      = file_get_contents( 'php://input' );
$header    = $_SERVER['HTTP_X_RED_HEADED_SIGNATURE'];
$expected  = 'sha256=' . hash_hmac( 'sha256', $body, $your_secret );
$is_valid  = hash_equals( $expected, $header );

Payload structure

{
  "event":        "export.delivered",
  "data":         { "job_id": 42, "profile_id": 7, "file": "orders-2026-05-29.csv" },
  "site":         "https://example.com",
  "timestamp_ms": 1780308600000,
  "plugin":       "red-headed-pro",
  "version":      "1.5.1"
}

Additional headers

HeaderValue
Content-Typeapplication/json; charset=UTF-8
User-AgentRed_Headed_Pro/{version}
X-Red_Headed_Pro-EventEvent name (e.g. export.delivered)
X-Red_Headed_Pro-SiteYour site URL
X-Red_Headed_Pro-Signaturesha256=... (if secret is set)

Troubleshooting

SymptomLikely causeFix
Export file is empty (only headers) Filters exclude every order — usually the date range or status list. Click 👁️ Preview on the profile. Widen the date range or tick more statuses.
SFTP: "authentication failed" Wrong password, wrong port, or the user lacks permission. Test credentials in a desktop SFTP client. Check the port (some servers use 2222, 2223…). Verify the remote folder exists and is writable.
Google Drive: 401 Unauthorized The access token expired (~1h lifespan). Generate a fresh token from Google's OAuth Playground.
Email sent but never received No SMTP configured — messages land in spam or are dropped. Install an SMTP plugin (e.g. WP Mail SMTP) with a transactional sender.
Email: "rate_limited" Lite edition: 30 emails per 24-hour sliding window. Upgrade to Pro for unlimited emails, or wait for the window to reset.
Schedule never fires WP cron only runs on page loads (low traffic = delays). Disable WP-Cron and call wp-cron.php from a system cron every minute.
XLSX: "ZipArchive missing" Host's PHP is missing the ZipArchive extension. Ask the host to enable ZipArchive, or switch format to CSV / JSON.
JSON has wrong data types Columns default to raw values (often strings in WooCommerce). Use per-column casts (int, number, money2) to force the right types.
Filename placeholders show as literal text Using wrong syntax (e.g. %order_id% instead of {order_id}). Use curly-brace syntax: {order_id}. See the placeholder reference.
REST endpoint returns 401/403 Auth type mismatch or expired credentials. Verify the auth type matches the API (Bearer vs Basic vs custom header) and refresh the token.
Local folder: "permission denied" The web server user lacks write access to the target directory. Check folder permissions (chmod 755 or 775). Use the "Create folder if missing" button in Settings.
Plugin not visible in admin menu The Froggy Hub is not installed or not activated. Install and activate the Hub first. Red-Headed appears under the Froggy Hub menu tree.
Pro features are locked License not activated or slug mismatch. Go to Froggy Hub → Licenses and verify your key is active for red-headed-pro.

Performance & limits

  • Order volume. Exports stream row by row. Tens of thousands of orders work. Hundreds of thousands take minutes — schedule those off-peak.
  • Memory. XLSX is the heaviest format (builds in memory). CSV, JSON, NDJSON are much lighter — prefer them for large exports.
  • HPOS. Fully compatible with WooCommerce High-Performance Order Storage. No configuration needed.
  • WPML / Polylang. Compatible. Order language is preserved.
  • Encryption. SFTP passwords and REST auth tokens are encrypted at rest with AES-256-CBC using wp_salt('auth').
  • Email rate limit. Lite: 30 emails per 24-hour sliding window. Pro: unlimited.
  • File retention. Files live in wp-content/uploads/lion-frog/exports/. Not auto-deleted — schedule periodic cleanup if disk space matters.
